# 🛒 E-commerce Core (Ядро интернет-магазина)

Этот проект представляет собой **ядро системы электронной коммерции**, реализующее базовые сущности интернет-магазина — такие как товары и категории — с возможностью загрузки данных из JSON и последующей работы с ними в различных интерфейсах (веб, Telegram-бот и т.д.).

## 📦 Функциональность

На текущем этапе реализованы:

- ✅ Абстрактные базовые классы `BaseProduct` и `BaseCategory` с обязательными методами.
- ✅ Класс `Product` — модель товара с полями: название, описание, цена (приватный атрибут с геттером и сеттером), количество.
- ✅ Миксин `PrintMixin` для вывода информации о создании объекта (автоматический print в конструкторе).
- ✅ Класс-метод `Product.new_product(data, products=None)`, который создаёт новый товар или обновляет существующий с таким же именем.
- ✅ Магический метод `__add__` в `Product`, возвращающий сумму стоимости двух товаров одного класса (с проверкой типа).
- ✅ Классы-наследники от `Product`:
  - `Smartphone` — с дополнительными атрибутами: эффективность, модель, память, цвет.
  - `LawnGrass` — с дополнительными атрибутами: страна-производитель, срок прорастания, цвет.
- ✅ Класс `Category` — модель категории с полями: название, описание, приватный список товаров `__products`.
- ✅ Подсчёт количества созданных категорий и товаров (через атрибуты класса `category_count` и `product_count`).
- ✅ Метод `Category.add_product(product)`, добавляющий товар в категорию с проверкой типа и обновлением счётчиков.
- ✅ Геттер `Category.products` для доступа к списку товаров.
- ✅ Метод `__str__` в `Product` и `Category`, возвращающий человеко-читаемое представление.
- ✅ Итератор `CategoryIterator` для перебора товаров в категории.
- ✅ Класс `Order` (наследник `BaseCategory`) с полями `product`, `quantity`, `total_price` и строковым представлением.
- ✅ Функции:
  - `read_json_file(path)` — загрузка данных из JSON-файла.
  - `create_objects_from_json(data)` — преобразование данных из JSON в объекты `Category` и `Product`.

## 🧪 Тестирование

Проект покрыт тестами с использованием `pytest`. В тестах реализовано:

- 🔁 Автоматический сброс счётчиков `category_count` и `product_count` перед каждым тестом.
- 🧪 Проверка инициализации объектов `Product`, `Category`, `Smartphone`, `LawnGrass`.
- 🧪 Проверка функции `create_objects_from_json()` — корректная трансформация JSON в объекты.
- 🧪 Проверка функции `read_json_file()` с использованием моков (`mock_open`, `patch`).
- 🧪 Проверка:
  - Геттера и сеттера для атрибута `price` в `Product` с проверкой запрета на нулевую/отрицательную цену.
  - Подтверждения понижения цены (с мокированием `input()`).
- 🧪 Проверка метода `__add__` в `Product`:
  - Корректное сложение товаров одного класса.
  - Возврат `NotImplemented` при сложении с несовместимым типом.
  - Выброс исключения `TypeError` при попытке сложить товары разных подклассов.
- 🧪 Проверка добавления товара в категорию с учётом типов и обновления счётчиков.
- 🧪 Проверка работы итератора `CategoryIterator`.
- 🧪 Проверка исключения `TypeError` при добавлении в категорию объекта не-наследника `Product`.
- 🧪 Проверка исключения `TypeError` при сложении разных подклассов товаров (например, `Smartphone` и `LawnGrass`).
- 🧪 Проверка класса `Order`:
  - Корректный расчёт общей стоимости заказа.
  - Корректное строковое представление заказа.

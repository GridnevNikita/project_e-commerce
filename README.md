# 🛒 E-commerce Core (Ядро интернет-магазина)

Этот проект представляет собой **ядро системы электронной коммерции**, реализующее базовые сущности интернет-магазина — такие как товары и категории — с возможностью загрузки данных из JSON и последующей работы с ними в различных интерфейсах (веб, Telegram-бот и т.д.).

## 📦 Функциональность

На текущем этапе реализованы:

- ✅ Класс `Product` — модель товара с полями: название, описание, цена, количество.
- ✅ Класс `Category` — модель категории с полями: название, описание, список товаров.
- ✅ Подсчёт количества созданных категорий и товаров (через атрибуты класса).
- ✅ Функция `read_json_file(path)` — загрузка данных из JSON-файла.
- ✅ Функция `create_objects_from_json(data)` — преобразование данных из JSON в объекты `Category` и `Product`.
- ✅ Приватный атрибут `__price` в классе `Product` с геттером и сеттером для контроля доступа и валидации цены.  
  - Сеттер запрещает установку нулевой или отрицательной цены.  
  - При попытке понизить цену срабатывает подтверждение через ввод с клавиатуры.
- ✅ Приватный список товаров `__products` в классе `Category`, доступный через геттер `products` (возвращает красиво отформатированную строку всех товаров).
- ✅ Метод `Category.add_product(product)` для добавления товара в категорию с автоматическим обновлением счётчика общего количества товаров.
- ✅ Класс-метод `Product.new_product(data, products=None)` для создания нового товара из словаря с обновлением количества и цены, если товар с таким именем уже есть.

## 🧪 Тестирование

Проект покрыт тестами с использованием `pytest`. Реализованы фикстуры и юнит-тесты для проверки:
- Инициализации объектов.
- Преобразования JSON в модели.
- Чтения JSON-файлов с использованием моков (`mock_open`, `patch`).
- Расширено тестовое покрытие с проверкой:  
  - Корректной работы геттера и сеттера для атрибута `price` в `Product` (включая валидацию и логику понижения цены).  
  - Добавления товара в категорию и корректного увеличения счетчиков.  
  - Класс-метода `new_product` для создания и обновления товара.  
  - Приватного доступа к списку товаров через геттер `products`.  

- Используются фикстуры для создания продуктов и категорий, а также `autouse` фикстура для сброса счетчиков перед каждым тестом.

- Мокирование функций чтения файлов и пользовательского ввода для тестирования сеттера цены.

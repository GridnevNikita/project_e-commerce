# 🛒 E-commerce Core (Ядро интернет-магазина)

Этот проект представляет собой **ядро системы электронной коммерции**, реализующее базовые сущности интернет-магазина — такие как товары и категории — с возможностью загрузки данных из JSON и последующей работы с ними в различных интерфейсах (веб, Telegram-бот и т.д.).

## 📦 Функциональность

На текущем этапе реализованы:

- ✅ Класс `Product` — модель товара с полями: название, описание, цена, количество.
- ✅ Класс `Category` — модель категории с полями: название, описание, список товаров.
- ✅ Подсчёт количества созданных категорий и товаров (через атрибуты класса).
- ✅ Функция `read_json_file(path)` — загрузка данных из JSON-файла.
- ✅ Функция `create_objects_from_json(data)` — преобразование данных из JSON в объекты `Category` и `Product`.
- ✅ Приватный атрибут `__price` в классе `Product` с геттером и сеттером:
  - Сеттер запрещает установку нулевой или отрицательной цены.
  - При понижении цены требуется подтверждение через пользовательский ввод.
- ✅ Приватный список `__products` в классе `Category`, доступный через геттер `products`, который возвращает список объектов `Product`.
- ✅ Метод `Category.add_product(product)` добавляет товар в категорию и увеличивает счётчик `product_count`.
  - В категорию можно добавлять только объекты класса `Product` или его наследников.
  - При добавлении любого другого объекта выбрасывается ошибка `TypeError`.
- ✅ Магический метод `__add__` в `Product`, возвращающий сумму стоимости (цена × количество) двух товаров.
  - Складывать можно только товары одного и того же класса.
  - При попытке сложить товары разных классов выбрасывается ошибка `TypeError`.
- ✅ Метод `__str__` в `Product` и `Category`, возвращающий человеко-читаемое представление объектов.
- ✅ Класс-метод `Product.new_product(data, products=None)`:
  - Создаёт новый товар из словаря.
  - Обновляет количество и цену существующего товара, если имя совпадает.
- ✅ Класс `CategoryIterator` — вспомогательный итератор, позволяющий перебирать товары одной категории в цикле `for`.
- ✅ Классы-наследники от `Product`:
  - `Smartphone` — расширен атрибутами: производительность (`efficiency`), модель (`model`), объём встроенной памяти (`memory`), цвет (`color`).
  - `LawnGrass` — расширен атрибутами: страна-производитель (`country`), срок прорастания (`germination_period`), цвет (`color`).

## 🧪 Тестирование

Проект покрыт тестами с использованием `pytest`. В тестах реализовано:

- 🔁 Автоматический сброс счётчиков `category_count` и `product_count` перед каждым тестом.
- 🧪 Проверка инициализации объектов `Product` и `Category`.
- 🧪 Проверка функции `create_objects_from_json()` — корректная трансформация JSON в объекты.
- 🧪 Проверка `read_json_file()` с использованием моков (`mock_open`, `patch`).
- 🧪 Проверка:
  - Геттера и сеттера для атрибута `price` (`Product`).
  - Подтверждения понижения цены (`input()` — мокируется).
  - Защиты от установки некорректной цены (нулевой/отрицательной).
  - Добавления товаров в категорию и обновления счётчиков.
  - Метод `__add__` у `Product`, включая проверку ограничения на одинаковый класс.
  - Поведения при попытке сложить `Product` с несовместимым типом (выбрасывается `TypeError`).
  - `Product.new_product()` — как при создании нового товара, так и при обновлении существующего.
- 🧪 Проверка работы `CategoryIterator`:
  - Итерация по товарам в категории.
  - Корректное завершение итерации (`StopIteration`).
- 🧪 Проверка добавления объекта не-наследника `Product` в категорию (выбрасывается `TypeError`).
- 🧪 Проверка корректной инициализации объектов `Smartphone` и `LawnGrass`, включая все новые атрибуты.
